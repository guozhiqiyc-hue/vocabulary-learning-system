# 🤖 AI自动补全工具使用指南

## 工具位置

```
D:\XX\WL\学习系统\02_English\词汇\每日词汇\vocabulary-learning-system-v6\auto-complete-tool.html
```

## 📋 功能说明

这个工具可以自动补全语块数据库中缺失的信息：
- ✅ 中文翻译 (meaning)
- ✅ 例句 (example)
- ✅ 关键词 (keywords)

## 🚀 使用步骤

### 1. 准备API密钥

#### 方案A：使用OpenAI (推荐)
- 访问 https://platform.openai.com/api-keys
- 创建API密钥
- 成本：GPT-3.5约 $0.002/1K tokens（非常便宜）
- 100个语块大约需要 $0.1-0.2

#### 方案B：使用Anthropic Claude
- 访问 https://console.anthropic.com/
- 创建API密钥
- 使用Claude Haiku模型最便宜

#### 方案C：使用国内API
- 支持任何OpenAI兼容的API
- 可以使用阿里通义千问、百度文心一言等
- 只需要提供API地址和密钥

### 2. 打开工具

在浏览器中打开 `auto-complete-tool.html`

### 3. 配置API

1. **选择AI服务提供商**：
   - OpenAI (GPT-4/GPT-3.5)
   - Anthropic (Claude)
   - OpenAI兼容接口

2. **输入API密钥**：
   - 粘贴你的API密钥
   - 密钥仅存储在浏览器本地，安全可靠

3. **配置模型**（可选）：
   - 默认使用 `gpt-3.5-turbo`（性价比最高）
   - 也可以使用 `gpt-4`（质量更好，价格稍高）

4. **设置批次大小**：
   - 建议 10-20（避免API限流）
   - 批次越小，越稳定，但速度稍慢

5. **选择补全选项**：
   - ☑️ 补全中文翻译
   - ☑️ 补全例句
   - ☑️ 补全关键词

### 4. 扫描数据

点击"📊 扫描数据"按钮，工具会：
- 扫描数据库中的所有语块
- 识别缺失信息的语块
- 显示统计信息

### 5. 开始处理

1. 点击"🚀 开始处理"按钮
2. 工具会自动批量处理不完整的语块
3. 实时显示处理进度和日志
4. 可以随时"暂停"或"继续"

### 6. 查看和应用结果

处理完成后：

1. **查看结果**：在"📊 结果"标签页查看所有补全的数据

2. **导出结果**：点击"📥 导出结果"保存为JSON文件

3. **应用到数据库**：点击"💾 应用到数据库"直接更新数据库

## 💡 使用技巧

### 降低成本
- 使用 GPT-3.5-turbo（性价比最高）
- 设置合理的批次大小（15-20）
- 先处理少量数据测试效果

### 提高质量
- 提供完整的语块格式（已有数据越完整，AI生成质量越好）
- 检查生成的例句是否正确标记关键词（用**包裹）
- 手动复核AI生成的内容

### 避免限流
- 批次大小不要太大（建议10-20）
- 每批之间自动延迟2秒
- 如果频繁限流，可以增大延迟

## 🛡️ 安全说明

- ✅ API密钥仅存储在浏览器本地存储
- ✅ 数据不会上传到任何第三方服务器
- ✅ 所有处理都在浏览器中完成
- ✅ 只会调用你配置的AI API

## ⚠️ 常见问题

### Q: 处理失败怎么办？
A: 检查以下几点：
- API密钥是否正确
- API余额是否充足
- 网络连接是否正常
- 模型名称是否正确

### Q: 生成的质量不满意？
A: 可以：
- 使用更高级的模型（如GPT-4）
- 手动编辑部分结果
- 重新单独处理某个语块

### Q: 如何验证结果？
A:
1. 先导出结果查看
2. 抽查几个语块验证质量
3. 确认无误后再应用到数据库

### Q: 会覆盖原有数据吗？
A: 不会！工具只会补全缺失的字段，已有数据不会修改。
