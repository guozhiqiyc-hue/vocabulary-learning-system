# 📚 教材数据导入指南

## 支持的教材数据格式

### ✅ 已支持格式

#### 1. 系统提取格式 (_system.json)
**文件示例**: `七上_system.json`, `七下_system.json`, `八上_system.json`, `八下_system.json`

**数据结构**:
```json
{
  "source": "七上.pdf",
  "total": 950,
  "cards": [
    {
      "id": "七上_001",
      "phrase": "be good at",
      "meaning": "",
      "example": "",
      "level": "核心词汇",
      "keywords": ["good", "at"],
      "unit": "Unit 1 New school, new friends, new start!",
      "type": "phrase"
    }
  ]
}
```

**说明**:
- `source`: 教材文件名
- `total`: 语块总数
- `cards`: 语块数组
- 自动处理缺失的 `meaning` 和 `example` 字段
- 自动识别主题单元

---

## 🚀 导入步骤

### 方法1: 通过系统界面导入

1. **打开学习系统**
   - 双击打开 `index-standalone.html`

2. **打开导入对话框**
   - 点击顶部工具栏的 **📦 导入/导出** 按钮

3. **选择文件**
   - 点击"选择文件"
   - 选择要导入的 JSON 文件（如 `七上_system.json`）

4. **选择导入模式**
   - **追加模式**: 保留现有语块，添加新语块（推荐）
   - **替换模式**: 清空现有语块，使用新数据

5. **开始导入**
   - 点击"导入数据"按钮
   - 等待导入完成

6. **查看结果**
   - 系统会显示导入统计：
     - 数据源
     - 原始数量
     - 已导入数量
     - 已跳过数量（重复数据）

## 💡 导入建议

### 首次导入
1. 使用 **追加模式** 导入第一本教材
2. 检查导入结果
3. 开始学习

### 多教材导入
1. 按顺序导入各册教材
2. 每次导入前可以创建备份
3. 使用追加模式避免数据丢失

### 数据管理
1. 定期创建完整备份
2. 删除不需要的重复语块
3. 使用编辑器修改释义和例句

---

## ⚠️ 注意事项

### 导入模式选择
- **追加模式**: 安全，不会丢失现有数据
- **替换模式**: 会清空所有现有语块，谨慎使用

### 数据处理
- 系统会自动跳过重复的语块
- 空白的 `meaning` 和 `example` 会自动填充
- 换行符会被自动清理

### 存储限制
- IndexedDB 通常有 50MB+ 的存储空间
- 950条语块大约占用 1-2MB
- 可以轻松导入多册教材

---

## 🔧 故障排除

### 导入失败
1. 检查文件格式是否正确（JSON格式）
2. 确认文件包含 `cards` 或 `phrases` 字段
3. 查看浏览器控制台错误信息

### 数据不完整
1. 原始数据可能包含空白数据
2. 系统会自动跳过无效数据
3. 可以在编辑器中手动补充

### 系统变慢
1. 大量数据可能影响性能
2. 建议分批导入（每册500-1000条）
3. 定期清理已掌握的旧语块
